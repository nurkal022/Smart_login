Системные требования

  • Node.js v18+ и npm
  • Python 3.10+
  • MetaMask — расширение для браузера
  • Браузер — Chrome, Firefox, Brave или Edge

Установка

Шаг 1. Клонирование репозитория
git clone <repo-url>
cd cripto
Шаг 2. Установка зависимостей Node.js
npm install
Шаг 3. Настройка Python-окружения
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

Развёртывание смарт-контракта

Шаг 1. Запуск локальной сети
npx hardhat node
# Сеть будет доступна на http://127.0.0.1:8545
# Chain ID: 31337
Шаг 2. Деплой контракта
npx hardhat run scripts/deploy.js --network localhost

Скопируйте адрес контракта из вывода — он понадобится для файла .env.

Шаг 3. Создание .env

Создайте файл backend/.env:

CONTRACT_ADDRESS=0x5FbDB...aa3
PRIVATE_KEY=0xac0974bec...2ff80
JWT_SECRET=ваш_секретный_ключ_32_символа
HARDHAT_RPC=http://127.0.0.1:8545
Важно: PRIVATE_KEY — это приватный ключ аккаунта-владельца контракта (Account #0 в Hardhat). Этот ключ используется бэкендом для записи событий в блокчейн.

Запуск системы

Необходимо запустить три компонента:

# Терминал 1 — блокчейн
npx hardhat node

# Терминал 2 — бэкенд
cd backend && source venv/bin/activate
source .env && uvicorn main:app --port 8001

# Терминал 3 — фронтенд
cd frontend && python3 -m http.server 5500

Настройка MetaMask

  1. Установите расширение MetaMask из магазина вашего браузера
  2. Создайте новый кошелёк или импортируйте существующий
  3. Добавьте сеть вручную: RPC URL: http://127.0.0.1:8545, Chain ID: 31337
  4. Для тестирования импортируйте Account #0 Hardhat по приватному ключу

Процесс входа

  1. Откройте http://localhost:5500/login.html
  2. Нажмите «Войти через MetaMask»
  3. MetaMask запросит подключение — подтвердите
  4. Подпишите сообщение с одноразовым кодом (nonce)
  5. Система проверит подпись и запишет вход в блокчейн
  6. Вы будете перенаправлены в личный кабинет

Панель администратора

Доступна только для владельца контракта. Возможности:

  • Добавление сотрудника — укажите имя и адрес кошелька (0x...)
  • Удаление сотрудника — мгновенное отключение доступа
  • Журнал аудита — все входы из блокчейна с временными метками
Важно: Все операции по добавлению и удалению сотрудников являются блокчейн-транзакциями. Они неизменны и видны всем участникам сети.

Смарт-контракт

Контракт EmployeeAuth.sol написан на Solidity 0.8.24 и содержит:

  • addEmployee(address, name) — регистрация сотрудника (onlyOwner)
  • removeEmployee(address) — удаление сотрудника (onlyOwner)
  • isEmployee(address) — проверка статуса (view)
  • logLogin(address) — запись входа (onlyOwner)
  • getLoginHistory() — получение журнала (view)
  • getEmployeeList() — список адресов (view)

Все мутации защищены модификатором onlyOwner и генерируют события (events) для off-chain мониторинга.
