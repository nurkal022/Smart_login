MetaMask Authentication

Self-Hosted Friendly

Deploy on your own infrastructure with a local Ethereum node. No dependency on third-party auth providers.

How It Works
  1. Employee opens the portal and clicks "Login with MetaMask"
  2. The system generates a unique one-time nonce
  3. Employee signs the nonce with their private key
  4. Backend recovers the address from the signature
  5. Smart contract confirms the address is a registered employee
  6. Login is recorded on-chain and a session token is issued
🔒

Security Model

The private key never leaves the employee's device. The server never sees a password. Even if the backend is fully compromised, there are no credentials to steal.


Backed by Ethereum cryptography (secp256k1 ECDSA)
